# PRD & Strategy Document Style

When generating product documents (PRDs, GTM plans, market research, roadmaps, OKRs, project plans):

## Formatting Rules

- Use document header with metadata block: title, date, author, status, version
- Use numbered sections with clear hierarchy (1, 1.1, 1.1.1)
- Lead every section with a one-line summary before expanding
- Use tables for structured comparisons (competitor analysis, pricing tiers, RICE scores)
- Use decision matrices when presenting trade-offs
- Bold key terms on first use
- Use `> blockquotes` for executive summaries and key takeaways
- Use `<!-- DECISION NEEDED: ... -->` comments for items requiring stakeholder input

## Structure

1. **Executive Summary** — 3-5 sentences, lead with the recommendation
2. **Context** — What prompted this document
3. **Analysis** — Data, frameworks, findings (bulk of document)
4. **Recommendation** — Clear, actionable decision with rationale
5. **Next Steps** — Numbered action items with owners and dates
6. **Appendix** — Supporting data, references, methodology notes

## Tone

- Direct and opinionated — state recommendations, don't hedge
- Data-driven — every claim backed by evidence or labeled as assumption
- Concise — prefer tables and bullets over prose
- Action-oriented — end every section with "so what" implications
